# 課堂打卡系統操作手冊

## 1. 系統目的

本系統提供教室現場使用的課堂打卡管理功能，包含：

- 學生出席打卡
- 管理者登入
- 課程維護
- 課堂時段維護
- Excel 出席匯出
- QR Code 打卡板

---

## 2. 系統需求

- Windows 環境
- 已安裝 `.NET 9 SDK`

---

## 3. 啟動方式

在專案根目錄執行：

```powershell
dotnet build
dotnet run
```

啟動後，瀏覽器開啟系統網址，例如：

```text
https://localhost:xxxx
```

---

## 4. 角色說明

### 4.1 一般使用者

一般使用者可：

- 查看課堂總覽
- 進入指定課堂打卡頁
- 輸入學號、姓名與備註完成打卡
- 使用 QR Code 進入打卡頁

### 4.2 管理者

管理者可：

- 登入後台
- 新增、編輯、刪除課程
- 新增、編輯、刪除課堂時段
- 開放或關閉打卡
- 匯出 Excel 出席紀錄
- 開啟 QR Code 打卡板

---

## 5. 首頁操作

首頁路徑：

```text
/
```

首頁可看到：

- 開放打卡課堂數
- 累積打卡筆數
- 每堂課的教師、教室、時間、主題
- 最近打卡紀錄

每張課堂卡片提供：

- `進入打卡頁`
- `QR Code`

若目前已登入管理者，還會額外看到：

- `開放打卡` 或 `關閉打卡`

---

## 6. 學生打卡流程

### 6.1 進入打卡頁

方式如下：

1. 從首頁點擊 `進入打卡頁`
2. 從 QR Code 掃碼進入

### 6.2 填寫資料

學生需輸入：

- 學號
- 姓名
- 備註（可不填）

### 6.3 送出結果

送出後系統會出現以下結果：

- `打卡成功，已完成出席登記。`
- 若同一堂課重複使用相同學號，系統會拒絕重複打卡
- 若課堂未開放，系統會提示該堂課無法打卡

### 6.4 防重複規則

同一堂課的唯一判定為：

- `課堂時段 + 學號`

同一學號在同一堂課只能成功打卡一次。

---

## 7. 管理者登入

登入頁路徑：

```text
/Account/Login
```

預設帳號：

```text
admin
```

預設密碼：

```text
ChangeMe123!
```

登入成功後會進入：

```text
/Admin
```

### 7.1 安全建議

正式環境請務必修改預設密碼，可於 `appsettings.json` 或環境變數設定：

```powershell
$env:AdminAuth__Username="admin"
$env:AdminAuth__Password="請改成正式密碼"
dotnet run
```

---

## 8. 管理後台操作

管理後台路徑：

```text
/Admin
```

後台分成兩個主要區塊：

- 課程維護
- 課堂時段維護

---

## 9. 課程維護

### 9.1 新增課程

在後台點選：

- `新增課程`

需填欄位：

- 課程代碼
- 課程名稱
- 教師姓名
- 教室
- 課程說明

按下 `儲存課程` 後即完成新增。

### 9.2 編輯課程

在課程列表點擊：

- `編輯`

修改後按下：

- `儲存課程`

### 9.3 刪除課程

在課程列表點擊：

- `刪除`

限制：

- 若該課程仍有課堂時段，系統不允許直接刪除課程
- 需先刪除該課程底下所有無出席紀錄的課堂

---

## 10. 課堂時段維護

### 10.1 新增課堂

在後台點選：

- `新增課堂`

需填欄位：

- 對應課程
- 課堂主題
- 開始時間
- 結束時間
- 是否立即開放打卡

按下 `儲存課堂` 即完成新增。

### 10.2 編輯課堂

在課堂列表點擊：

- `編輯`

可修改：

- 所屬課程
- 主題
- 開始時間
- 結束時間
- 是否開放打卡

### 10.3 刪除課堂

在課堂列表點擊：

- `刪除`

限制：

- 若此課堂已有打卡紀錄，系統不允許刪除
- 目的是避免出席資料遺失

### 10.4 開放或關閉打卡

管理者可於以下位置切換打卡狀態：

- 首頁課堂卡片
- 管理後台課堂列表

狀態說明：

- `開放中`：學生可打卡
- `關閉中`：學生不可打卡

---

## 11. Excel 出席匯出

在管理後台課堂列表中，選擇指定課堂後點擊：

- `Excel`

下載內容包含：

- 課程代碼
- 課程名稱
- 課堂主題
- 課堂時間
- 學號
- 姓名
- 打卡時間
- 備註

匯出格式：

```text
.xlsx
```

---

## 12. QR Code 打卡板

在首頁或管理後台可進入 QR Code 打卡板。

功能包含：

- 顯示該堂課 QR Code
- 顯示對應打卡網址
- 可直接開啟打卡頁
- 可列印投影使用

適用情境：

- 教室投影
- 列印貼在教室門口
- 現場掃碼打卡

---

## 13. 資料儲存位置

系統資料檔案：

```text
App_Data/attendance-data.json
```

內容包含：

- 課程資料
- 課堂時段資料
- 出席打卡紀錄

注意事項：

- 首次啟動若沒有資料檔，系統會自動建立種子資料
- 建議定期備份此檔案

---

## 14. 常見問題

### 14.1 為什麼學生無法打卡？

請確認：

- 該課堂是否存在
- 該課堂是否已開放打卡
- 是否已經用同一學號打過卡

### 14.2 為什麼不能刪除課堂？

因為該課堂已存在打卡紀錄，系統為避免資料遺失而禁止刪除。

### 14.3 為什麼不能刪除課程？

因為該課程底下仍有課堂時段，必須先處理課堂資料。

### 14.4 管理者密碼在哪裡改？

目前需手動修改：

- `appsettings.json`
- `appsettings.Development.json`
- 或環境變數 `AdminAuth__Password`

---

## 15. 建議操作流程

### 教務人員每日使用流程

1. 管理者登入
2. 建立今日課堂時段
3. 開啟對應課堂打卡
4. 投影或列印 QR Code 打卡板
5. 課後匯出 Excel 出席紀錄
6. 關閉打卡
