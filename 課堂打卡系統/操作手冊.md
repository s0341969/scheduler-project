# 課堂打卡系統操作手冊

## 1. 系統目的

本系統提供教室現場使用的課堂打卡管理功能，包含：

- 學生本人登入後打卡
- 管理者登入
- 課程維護
- 課堂時段維護
- Excel 出席匯出
- QR Code 打卡板
- 可疑打卡複核

---

## 2. 系統需求

- Windows 環境
- 已安裝 `.NET 9 SDK`

---

## 3. 啟動方式

在專案根目錄執行：

```powershell
dotnet build
dotnet run
```

啟動後，瀏覽器開啟系統網址，例如：

```text
https://localhost:xxxx
```

---

## 4. 角色說明

### 4.1 學生

學生可：

- 先以學生帳號登入
- 從課堂 QR Code 進入打卡頁
- 以登入身分完成打卡

### 4.2 管理者

管理者可：

- 登入後台
- 新增、編輯、刪除課程
- 新增、編輯、刪除課堂時段
- 開放或關閉打卡
- 匯出 Excel 出席紀錄
- 開啟 QR Code 打卡板
- 檢視可疑打卡紀錄

---

## 5. 首頁操作

首頁路徑：

```text
/
```

首頁可看到：

- 開放打卡課堂數
- 累積打卡筆數
- 每堂課的教師、教室、時間、主題
- 最近打卡紀錄

每張課堂卡片提供：

- `開啟 QR 打卡板`
- `學生登入`

若目前已登入管理者，還會額外看到：

- `開放打卡` 或 `關閉打卡`

---

## 6. 學生打卡流程

### 6.1 學生登入

登入頁路徑：

```text
/Student/Login
```

目前示範帳號：

- `S1123001 / Student123!`
- `S1123002 / Student123!`
- `S1123003 / Student123!`

### 6.2 進入打卡頁

方式如下：

1. 先登入學生帳號
2. 連上學校班級 SSID
3. 掃描課堂 QR Code
4. 進入打卡頁

### 6.3 填寫資料

學生登入後系統會自動帶入：

- 學號
- 姓名

學生僅需輸入：

- 備註（可不填）

### 6.4 送出結果

送出後系統會出現以下結果：

- `打卡成功，已完成本人出席登記。`
- 若同一堂課重複打卡，系統會拒絕
- 若 QR Code 已失效，系統會要求重新掃描
- 若來源網路不在班級 SSID 網段，系統會依設定拒絕或標示可疑

### 6.5 本人驗證方式

目前系統的本人驗證依據為：

- 學生帳號登入
- 動態 QR Code 時效驗證
- 班級 SSID 對應網段驗證
- 裝置指紋與來源資訊留存

注意：

- Web 系統無法直接讀取手機目前連到的 SSID 名稱
- 系統實際上是透過來源 IP 是否落在班級 SSID 的 DHCP 網段來判定

### 6.6 防重複規則

同一堂課的唯一判定為：

- `課堂時段 + 學號`

同一學號在同一堂課只能成功打卡一次。

## 7. 管理者登入

登入頁路徑：

```text
/Account/Login
```

預設帳號：

```text
admin
```

預設密碼：

```text
ChangeMe123!
```

登入成功後會進入：

```text
/Admin
```

### 7.1 安全建議

正式環境請務必修改預設密碼，可於 `appsettings.json` 或環境變數設定：

```powershell
$env:AdminAuth__Username="admin"
$env:AdminAuth__Password="請改成正式密碼"
dotnet run
```

---

## 8. 管理後台操作

管理後台路徑：

```text
/Admin
```

後台分成三個主要區塊：

- 課程維護
- 課堂時段維護
- 最近可疑打卡

---

## 9. 課程維護

### 9.1 新增課程

在後台點選：

- `新增課程`

需填欄位：

- 課程代碼
- 課程名稱
- 教師姓名
- 教室
- 課程說明

按下 `儲存課程` 後即完成新增。

### 9.2 編輯課程

在課程列表點擊：

- `編輯`

修改後按下：

- `儲存課程`

### 9.3 刪除課程

在課程列表點擊：

- `刪除`

限制：

- 若該課程仍有課堂時段，系統不允許直接刪除課程
- 需先刪除該課程底下所有無出席紀錄的課堂

---

## 10. 課堂時段維護

### 10.1 新增課堂

在後台點選：

- `新增課堂`

需填欄位：

- 對應課程
- 課堂主題
- 開始時間
- 結束時間
- 是否立即開放打卡

按下 `儲存課堂` 即完成新增。

### 10.2 編輯課堂

在課堂列表點擊：

- `編輯`

可修改：

- 所屬課程
- 主題
- 開始時間
- 結束時間
- 是否開放打卡

### 10.3 刪除課堂

在課堂列表點擊：

- `刪除`

限制：

- 若此課堂已有打卡紀錄，系統不允許刪除
- 目的是避免出席資料遺失

### 10.4 開放或關閉打卡

管理者可於以下位置切換打卡狀態：

- 首頁課堂卡片
- 管理後台課堂列表

狀態說明：

- `開放中`：學生可打卡
- `關閉中`：學生不可打卡

### 10.5 QR Code 時效

- 每次開啟 QR Code 打卡板都會產生新的打卡 Token
- 預設有效時間為 5 分鐘
- QR Code 打卡板會自動刷新，確保畫面上的 QR Code 保持有效

下載內容包含：

- 課程代碼
- 課程名稱
- 課堂主題
- 課堂時間
- 學號
- 姓名
- 打卡時間
- IP
- 裝置指紋
- 網段狀態
- 可疑原因
- 備註

匯出格式：

```text
.xlsx
```

## 12. QR Code 打卡板

在首頁或管理後台可進入 QR Code 打卡板。

功能包含：

- 顯示該堂課 QR Code
- 顯示對應打卡網址
- 顯示 QR 有效截止時間
- 自動刷新 QR Code
- 可列印投影使用

適用情境：

- 教室投影
- 列印貼在教室門口
- 現場掃碼打卡

## 14. 班級 SSID 驗證設定

主要設定如下：

- `AttendanceSecurity__AllowedIpPrefixes`
- `AttendanceSecurity__ExpectedSsidLabel`
- `AttendanceSecurity__StrictNetworkValidation`

範例：

```json
"AttendanceSecurity": {
  "QrTokenLifetimeMinutes": 5,
  "StrictNetworkValidation": false,
  "ExpectedSsidLabel": "學校班級 SSID",
  "AllowedIpPrefixes": [ "192.168.50.", "10.50." ]
}
```

說明：

- `AllowedIpPrefixes` 要改成你們班級 SSID 真正發出的 IP 網段
- `StrictNetworkValidation=false` 時，不在網段內仍可打卡，但會列為可疑
- `StrictNetworkValidation=true` 時，不在網段內直接拒絕打卡

## 15. 常見問題

### 15.1 為什麼學生無法打卡？

請確認：

- 是否已先登入學生帳號
- 是否掃描最新 QR Code
- QR 是否已過期
- 是否已經用同一學號打過卡
- 是否已連上班級 SSID 對應網段

### 15.2 為什麼某筆紀錄會被標示為可疑？

常見原因：

- 來源 IP 不在允許的班級 SSID 網段
- 同一裝置近期曾被不同學號使用
- 缺少正常瀏覽器識別資訊

### 15.3 為什麼不能刪除課堂？

因為該課堂已存在打卡紀錄，系統為避免資料遺失而禁止刪除。

### 15.4 為什麼不能刪除課程？

因為該課程底下仍有課堂時段，必須先處理課堂資料。

### 15.5 管理者密碼在哪裡改？

目前需手動修改：

- `appsettings.json`
- `appsettings.Development.json`
- 或環境變數 `AdminAuth__Password`

### 15.6 班級 SSID 驗證在哪裡改？

目前需修改：

- `AttendanceSecurity__AllowedIpPrefixes`
- `AttendanceSecurity__ExpectedSsidLabel`
- `AttendanceSecurity__StrictNetworkValidation`

## 16. 建議操作流程

### 教務人員每日使用流程

1. 管理者登入
2. 建立今日課堂時段
3. 開啟對應課堂打卡
4. 確認班級 SSID 網段設定正確
5. 投影 QR Code 打卡板
6. 課後查看可疑紀錄
7. 匯出 Excel 出席紀錄
8. 關閉打卡
