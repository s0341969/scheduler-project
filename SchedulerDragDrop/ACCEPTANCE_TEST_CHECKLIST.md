# 手動驗收測試清單（Manual Acceptance Checklist）

> 專案：SchedulerDragDrop（C# WPF）
> 目的：逐項確認目前所有需求與新功能皆符合預期。
> 測試環境：Windows + .NET 8 SDK

## 測試前置

- [ ] 在專案目錄執行：`dotnet build`，確認成功（0 errors）。
- [ ] 啟動程式：`dotnet run`。
- [ ] 畫面成功開啟，沒有啟動例外。

## A. 介面與語系

1. 繁體中文 UI
- [ ] 檢查主畫面標題、按鈕、提示、右鍵選單、錯誤訊息皆為繁體中文。
- 預期：不出現英文殘留（除專有名詞如 Job/Seq）。

2. 版面配置
- [ ] 確認上方為機台欄、左側為時間軸、中央為排程區、右側為選取製程資訊。
- 預期：版面與需求一致。

## B. 排程與資料來源

3. 資料庫模式（有設定環境變數時）
- [ ] 設定 `PROD_SCHEDULER_CONN` 與資料表名稱後啟動程式。
- [ ] 確認資料來自資料庫（製卡編號/機台與示範資料不同）。
- 預期：畫面資料反映資料庫 table 內容。

4. 回退示範資料模式（未設定或連線失敗）
- [ ] 清除 `PROD_SCHEDULER_CONN` 後啟動程式。
- [ ] 確認有 30 張製卡（CARD-001 ~ CARD-030）。
- [ ] 每張製卡有 5~10 製程。
- [ ] 機台總數為 15 台。
- 預期：可自動回退，資料量與需求一致。

5. 自動排程
- [ ] 點擊「自動排程」。
- 預期：可產生排程，區塊有開始/結束時間。

## C. 時間軸與視覺

6. 時間比例顯示
- [ ] 觀察區塊垂直位置是否對應開始時間。
- [ ] 觀察區塊高度是否對應工時長短。
- 預期：區塊位置/大小與時間軸一致。

7. 非工作時段底色
- [ ] 檢查 08:00~17:00 以外時段。
- 預期：非工作時段顯示灰底（時間軸與機台區同時生效）。

8. 縮放
- [ ] 拉動縮放滑桿（最小/中間/最大）。
- 預期：時間軸與區塊同步縮放，排程相對位置不錯亂。

9. 小區塊資訊顯示
- [ ] 將滑鼠停在小區塊上。
- 預期：顯示 Tooltip（標題、Meta、開始、結束）。

## D. 選取與高亮

10. 單選
- [ ] 單擊任一製程。
- 預期：只有該製程被選取；主選製程有額外外框色。

11. 多選（Ctrl+點擊）
- [ ] 按住 Ctrl 依序點 2~3 個製程。
- 預期：可同時維持多個選取。

12. 取消多選
- [ ] 按住 Ctrl 再點已選製程。
- 預期：該製程取消選取。

13. 清空選取
- [ ] 點機台空白處。
- 預期：所有選取清空。

14. 同製卡關聯高亮
- [ ] 點任一製程。
- 預期：同一張製卡其他製程顯示關聯高亮。

15. 點選不跳動
- [ ] 單擊多次不同製程。
- 預期：畫面不會自動跳動或捲動。

## E. 拖拉行為

16. 可拖至任意機台
- [ ] 將製程拖到不同類型機台。
- 預期：允許拖放，不因機台類型被拒絕。

17. 落點時間放置
- [ ] 在同一機台不同高度放置同一製程。
- 預期：依滑鼠落點時間放置，不自動貼齊上方最後工件尾端。

18. 目標機台撞期後推
- [ ] 把製程放到已被占用時段。
- 預期：該時段內與其後工件自動後推。

19. 來源機台不前推
- [ ] 將某製程跨機台拖走。
- 預期：來源機台原本在其後的工件不前推（不早於原開始時間）。

20. 製程序列防呆
- [ ] 嘗試把同製卡後段製程排到前段之前。
- 預期：操作被拒絕並顯示製程序衝突訊息。

## F. 右鍵推移

21. 安全流程
- [ ] 右鍵製程。
- 預期：先看到二層選單，不會直接執行推移。

22. 開啟推移設定
- [ ] 點「推移選取項目 -> 開啟推移設定...」。
- 預期：先出現確認視窗；按「是」才開啟日期時段對話框。

23. 只推移選取物件
- [ ] 先多選部分製程，執行日期時段推移。
- 預期：僅選取製程被推移，不會整張製卡一起移。

24. 未選取區塊保護
- [ ] 嘗試把選取製程往上移到會牽動未選取區塊的位置。
- 預期：系統自動回復並提示「未選取區塊發生位移，已取消本次移動」。

## G. 按鈕推移（±1 小時）

25. 選取項目 -1 小時
- [ ] 選取 1~2 個製程，按「選取項目 -1 小時」。
- 預期：僅選取製程提前 1 小時；若牽動未選取區塊則取消並提示。

26. 選取項目 +1 小時
- [ ] 選取 1~2 個製程，按「選取項目 +1 小時」。
- 預期：僅選取製程延後 1 小時；若牽動未選取區塊則取消並提示。

## H. 文件一致性

27. README 一致
- [ ] 檢查 `README.md` 功能描述與實際行為一致。

28. CHANGELOG 有時間
- [ ] 檢查 `CHANGELOG.md` 每筆變更是否含時間戳。

29. TODO 更新
- [ ] 檢查 `TODO.md` 是否包含目前待辦。
